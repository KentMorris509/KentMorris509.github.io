Projects

Chess Board Game

Project Motivation: The Motivation for this project was out of the interest to make a movable GUI application of a beloved game that I like to play. This was also interesting because in order to build a chess game you need to utilize the skills of OOP and understanding all the use cases that are possible in the game.

Functional Requirements: the requirements of the game were to make the game user friendly with a dragable game board that restricts illegal movement and highlights valid moves that the user can make. The project also plays by the exact rules of chess with check rules, end pond rules, and casteling. Having all these requirements is important to make sure that the game is functional and user-friendly.

Project Design: The design of the project was followed using abstract classes and interfaces for the board and individual piece that are connected to a GUI interface as the back end logic.
